Seesaw Island Riddle

A small text-based game which allows the user to play out the seesaw island riddle from Brooklyn 99.

Repository

Background

This program is a text-based game which allows the user to play out the seesaw island riddle from Brooklyn 99, paraphrased below.

There are 12 people on an island. There is a seesaw on the island. 11 of the people weigh the exact same amount. The other person has a slightly different weight. The user must determine who the person of unequal weight is using only the seesaw, and the seesaw can only be used 3 times.

I heard the riddle on the show and became a little obsessed with it. I spent much of my spare time thinking about the riddle, and avoided any and all internet searches which might reveal the answer. I really wanted the satisfaction of having solved it for myself.

Once I had an algorithm I thought worked, I wrote this program to give myself a nice way to test it.

NOTE: The balance computation does not take into account the distance of the people from the fulcrum of the seesaw. It is computed as if all people present on the seesaw are equidistant from the fulcrum. One of my original solutions was dependent on observing the speed with which the seesaw fell. I carefully asked an AI chatbot if the solution to the riddle depended on this, and was told the accepted solutions do not. Since the velocity of the seesaw does not matter, the exact position of each person does not either, only what side they are on.

How to Play

The game is played in a terminal. $ make play will build and start the game for you.

When the game is first started, an screen explaining the instructions will appear. You can enter ‘M’ at any time to see this screen again. Here is a summary:

  • ‘A9’ -> Move islander A to postion 9.
  • ‘R’ -> Remove all islanders from the seesaw.
  • ‘T’ -> Test the current seesaw setup. This is one of your three seesaw uses.
  • ‘W’ -> Discover who is the special islander.
  • ‘Q’ -> Quit the game.
  • ‘M’ -> Display instructions message.

Each time you run the game, the special islander is selected at random, and the weight difference is random as well.

NOTE: This program was written on a linux machine, and has only been played on Ubuntu and MacOS. The Makefile is not written for Windows, and there may be graphical glitches on other operating systems related to clearing the screen.

My Algorithm

My algorithm is based on a few ideas

  1. It is desirable to use more islanders per operation than fewer, but not desirable to use all the islanders in a single operation.
  2. Each time an islander is placed on the seesaw, additional knowledge about that islander may be gained.
  3. Different arrangements of the same set of islanders can produce different outcomes.
  4. Islanders which have been proven to have the standard weight may be used to test against unknown islanders.

Branching Tree

In this tree, the islanders will have names A, B, C, D, E, F, G, H, I.

The words "safe individual" will be used to describe an islander which has been shown already to not be the source of imbalance.

The branching logic for the algorithm is slightly too large to be read in one table, and the initial two branches formed from the outcome of the first test are quite different, so I will show the first test here, and then two tables will follow, one for each branch.

Test 1

Test 1 will always be a 4v4.

This follows idea (1.). 4v4 is the right choice here rather than 5v5, because the outcome of the 5v5 could leave you with 10 potential imbalances and only two tests remaining, which is not solvable. Likewise, 6v6 is not advisable, as it will not reveal anything you do not already know. You already know the imbalance is somewhere in the set of 12.

ABCD vs EFGH

If Test 1 is balanced, the imbalance is in the set IJKL and will be found in table 1.

If Test 1 is unbalanced, the imbalance is in the set ABCDEFGH and will be found in table 2.

Table 1 -- IJKL contains imbalance.

Pre-test 2 explainer

If Test 1 was balanced, the imbalance lies in the set IJKL.

Take 3 of the 4 potential, add a safe individual, and test.

This decision pulls from ideas (1.) and (4.)
Test 2

IJ vs KA
Pre-test 3 explainer

If Test 2 was balanced, the imbalance is in the untested islander, L.

If Test 2 was unbalanced, the imbalance is in the set IJK.

Take the 2 which have been sharing a side so far, and test them against each other.
Test 3

N/A

I vs J
Outcome

Imbalance in L

If Test 3 was balanced, the imbalance is in K.

If Test 3 was unbalanced, the imbalance can be determined from comparing the current outcome with the previous tests where I and J were on the seesaw. This is idea (2.).

In Test 2 (IJ vs KA) if the RIGHT half fell toward the ground, you know IJ is light or KA is heavy. In Test 3 (I vs J), if the RIGHT half falls, it is therefore because I is light. Likewise, if the LEFT half falls, it is because J is light. The same logic can be applied if Test 2 has the LEFT half falling. In this case, instead of I or J being light, they must be heavy.

Table 2 -- ABCDEFGH contains imbalance.

Pre-test 2 explainer

If Test 1 was unbalanced, the imbalance lies in the set ABCDEFGH.

Take 5 of the 8 potential, reverse the positions of 2, and add a safe individual.

By reversing the positions of 2 islanders, we are implementing idea(3).
Test 2

AFC vs EBI
Pre-test 3 explainer

If Test 2 was balanced, the imbalance is in the untested set, DGH.

Take the two from this set which have shared a side and test them against each other.

If Test 2 was unbalanced, the imbalance is in the set AFCEB.

By comparing the outcome of Test 2 to Test 1, we can determine whether the imbalance is in the set ACE or FB. If the seesaw side which fell is the same from Test 1 to Test 2, we know the imbalance did not change sides, and therefore lies in set ACE. If the falling seesaw side in Test 2 is opposite Test 1, we know the imbalance changed sides, and is therefore in set FB. Based on this, the use should either Test one from FB against a safe individual, or split the two sharing a side from ACE, and test them against each other.

Note here, this particular outcome is why we did not use 6 of the 8 potential islanders when performing Test 2. If we had done that, a possible set containing the imbalanced islander would be ACEG. It is not possible to determine the source of the imbalance from four possibilities with one remaining test.
Test 3

G vs H

F vs I

A vs C
Outcome

If Test 3 was balanced, the imbalance is in D.

If Test 3 was unbalanced, the imbalance can be determined from comparing the current outcome with the previous tests where G and H were on the seesaw. The method is the same as the Test 3 unbalanced outcome in Table 1.

If Test 3 was balanced, the imbalance is in B.

If Test 3 was unbalanced, the imbalance is in F.

If Test 3 was balanced, the imbalance is in E.

If Test 3 was unbalanced, the imbalance can be determined from comparing the current outcome with the previous tests where A and C were on the seesaw. The method is the same as the Test 3 unbalanced outcome in Table 1.

Example from the Game

The branching logic can be hard to follow without showing results. The following is an example which follows the rightmost path in Table 2.

Test 1

SEESAW: ABCD -^- EFGH

The RIGHT half of the seesaw drops toward the ground.

From this, we know the imbalanced islander is in the set ABCDEFGH.

Test 2

SEESAW: AFC -^- EBI

The RIGHT half of the seesaw drops toward the ground.

From this, we know the imbalanced islander did not change sides, so must be in set ACE.

Test 3

SEESAW: A -^- C

The LEFT half of the seesaw drops toward the ground.

From the result of Test 2, we know AC is light or E is heavy. Since this test was not balanced, we know A or C is the imbalanced islander. Since there is no remaining possibility that one of them is heavy, the LEFT half dropping shows us that C is light.
