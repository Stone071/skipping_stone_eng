Audio Bandpass Filter

Revising a three band audio equalizer from class

Background

A lab project we performed early in undergrad was implementing a 3 band audio equalizer. We were advised to use a multiple-feedback bandpass filter for at least one stage of the amplifier, and given specifications in the form of center frequency, quality factor, and gain for each channel.

That lab flew by, and I was so floored when the equalizer worked that I didn't stop to question some aspects of its performance which could have been much better. Here, I'll be taking a closer look at the most interesting portion of the equalizer which we designed, which is the bandpass filter. I'll be revising the design in favor of cascaded high-pass and low-pass filters, and making it clear why.

Requirements

The Given Specifications

Center Frequency Quality Factor Gain
Bass 200 Hz +- 10% 0.7 +- 10% 0.4 - 8
Mid 1100 Hz +- 10% 0.7 +- 10% 0.2 - 4
Treble 6000 Hz +- 10% 0.7 +- 10% 0.1 - 2

The center frequency and gain are intuitive requirements. Each channel must target a specific passband of frequencies, and the gain needs to be adjustable so the user can notice the difference. The quality factor given was not immediately intuitive. Quality factor is a measure how much a system damps oscillations. A system with a low quality factor (Q < 1/2) is overdamped, and will not oscillate. A system with a high quality factor (Q > 1/2) is underdamped, and will oscillate when excited by a specific frequency, and will continue oscillating longer depending on how high the quality factor is. A system Quality factor Q = 1/2 is critically damped, and will not oscillate, but also remains quickly responsive to sharp inputs. In a bandpass filter, quality factor has an effect on bandwidth as well, with high quality factors producing a more narrowband, peaked response. By selecting 0.7 as the quality factor, we were attempting to produce a flatter response in the passband, and not have a strong peak at the center frequency.

The Missed Requirement

When designing for audio and many other applications, it is desirable to amplify selected frequencies evenly across the entire bandwidth. You effectively want to take a rectangular window of frequencies and boost them all the same amount. Unfortunately, this is not what was achieved with the equalizer designed in lab. Above is a frequency response plot from the output of the 3 channel equalizer.

It is very clear in this frequency response that the frequencies in between the center frequency of each bandpass filter are not being effectively amplified. The amplitude which is achieved at center frequency is not being achieved throughout the passband of each filter. There are a couple things contributing to this. Firstly, the bandpass topology used here is not the best choice. Multiple-feedback bandpass filters are good for high Q, low bandwidth applications, selectively boosting a very narrow frequency band.

More importantly, a second order bandpass filter can never achieve this wide flat response in the passband by the basic fact that the relationship of the output to the input has a second-order expression in the denominator. Anyone can see this by just graphing any second order function.

We can pretend in this plot that frequency is on the x axis, and filter response is on the y axis. The truth is that the response of a second order bandpass filter will always look quite parabolic. It is a fundamental characteristic of the design. The actual transfer function for the topology implemented is

\(\frac{-sCR_{2}}{C^{2}R_{1}R_{2}s^{2} + 2CR_{1}s + 1} = \frac{V_{out}}{V_{in}}\)

The derivation can be found in Appendix A.

Design

Old

The old design was a second order multiple-feedback bandpass filter. As discussed above, it cannot achieve the flat response across the passband which we desire.

New

A very simple solution for creating a passband of any width while having a flat response in the middle frequencies is to cascade a high-pass and a low-pass filter together. The high-pass removes low frequencies, the low-pass removes high frequencies, and we are left with a window of unchanged response in the middle. This concept is illustrated simply by these three plots below.

In the final plot, one can see that that by combining the left region of the left plot and the right region of the right plot, we have created a zone in the middle where they overlap. This middle zone is the passband. For now, we'll consider simpler better, and pursue this idea with some first order high and low pass filters.

Circuit Layout

Topology

The topology being employed here is very simple. In the lowpass filter, placing a capacitor in the feedback path causes the impedance seen by the feedback to be reduced as frequency increases, thus causing more feedback and reducing gain. For the highpass filter, the capacitor in the signal path to the input poses high impedance at low frequency, effectively blocking low frequencies.

Design Equations

The low-pass filter has a transfer function of

\( -\frac{R_2}{R_1}\ \frac{1}{s R_2 C + 1} = \frac{V_{out}}{V_{in}} \)

The derivation of this can be found in Appendix A.

This transfer function tells us some things implicitly. We can see that the passband has a gain of -(R2/R1). We can also see that the first order pole will exist when sR2C = -1. This is our cut off frequency.

\(\frac{1}{C 2 \pi R_{2}} = F_{o}\)

In order to make a capacitor selection for a given cut off frequency, this equation should be solved for C.

\(\frac{1}{F_o 2 \pi R_{2}} = C\)

The high-pass filter has a transfer function of

\(- \frac{R_{2}}{R_{1}} \frac{R_{1} C s}{R_{1} C s + 1} = \frac{V_{out}}{V_{in}}\)

The derivation of this can be found in Appendix A.

Like the low-pass filter, this function tells us the passband gain is

\(- \frac{R_{2}}{R_{1}} = H_{o}\)

and the cut-off frequency is

\(\frac{1}{C 2 \pi R_{1}} = F_{o}\)

Simulation

The simulation quickly revealed something disappointing, which is that the passbands required for audio filters are very narrow, in many cases only one decade in width. This is narrow enough that when using low order filters, like the first order filters used here, the sloping shoulder of each filter meets in the middle, and produces an amplitude peak in the middle of the passband, with amplitude reducing immediately on either side of the center. This can be seen below using the component values for the high channel.

In order to really see the flat passband of these cascaded filters, the bandwidth of must be much wider. See below, where bandwidth is almost 300 kHz.

Further Improvements

I retract what I said about this being much better accomplished with cascading first order filters. The first order filters have a -20dB/decade slope, which turns out to be far too shallow for making effective audio filters. 0-200Hz, 300-3000Hz, and even 3000-20000Hz are quite narrow passbands, and if we want these three passbands to have a flatter response across the curve while not overlapping in a significant way, we need to use a higher order filter to have a sharper drop in gain at the cutoff frequencies.

For example, here is a preliminary result for cascading highpass and lowpass second order filters, both designed with Sallen-Key topology for maximally-flat butterworth response, having Q approximately 0.707. This page will be updated soon to discuss this design choice instead.

Regardless of first or second-order filters, one great thing about the choice to use cascaded high and low-pass filters rather than a monolithic band-pass filter is that there is more control able to be exercised over the parameters dictating the specific response at each end of the passband, due to component value changes controlling the low corner frequency not affecting the filter controlling the high corner frequency, and vice versa. Of course, more components costs more money.

Appendix A: Solutions for Transfer Functions and System Characteristics

Multiple Feedback Bandpass Filter

Node Voltage Method

Node 1, between R1 and C

\(\frac{V_{1} - V_{in}}{R_{1}} + \frac{V_{1} - V_{out}}{\frac{1}{C s}} + \frac{V_{1}}{\frac{1}{C s}} = 0\)

\(\frac{V_{1}}{R_{1}} - \frac{V_{in}}{R_{1}} + C V_{1} s + - C V_{out} s + C V_{1} s = 0\)

Node 2, at the inverting input

\(- \frac{V_{out}}{R_{2}} - \frac{V_{1}}{\frac{1}{Cs}} = 0\)

\(- \frac{V_{out}}{R_{2}} - C V_{1} s = 0\)

\( - \frac{V_{out}}{C R_{2} s} = V_{1}\)

Substituting Node 2 equation into Node 1 equation

\(-\frac{\frac{V_{out}}{CR_{2}s}}{R_{1}} - \frac{V_{in}}{R_{1}} - \frac{CV_{out}s}{CR_{2}s} - CV_{out}s - \frac{CV_{out}s}{CR_{2}s} = 0\)

\(\frac{-V_{out}}{C s R_{1} R_{2}} - \frac{V_{in}}{R_{1}} - \frac{2V_{out}}{R_{2}} - CV_{out}s = 0\)

\(- V_{out}\left( \frac{1}{C s R_{1} R_{2}} + \frac{2}{R_{2}} + Cs \right) = \frac{V_{in}}{R_{1}}\)

\(- V_{out} \left( \frac{R_{1}}{C s R_{1} R_{2}} + \frac{2 R_{1}}{R_{2}} + CR_{1}s\right) = V_{in}\)

\(- V_{out} \left(\frac{1}{C R_{2} s} + \frac{C 2 R_{1} s}{C R_{2} s} + \frac{C^{2} s^{2} R_{1} R_{2}}{C R_{2} s}\right) = V_{in}\)

Final Transfer Function

\(\frac{-sCR_{2}}{C^{2}R_{1}R_{2}s^{2} + 2CR_{1}s + 1} = \frac{V_{out}}{V_{in}}\)

2nd order transfer function general form

\(\frac{K \omega_{n}^{2}}{s^{2} + 2\zeta\omega_{n}s + \omega_{n}^{2}} = H\)

Placing the transfer function in the general form, we can relate the component values to the characteristics of the second order system.

Fundamental Frequency

\(\frac{1}{C \sqrt{R_{1} R_{2}}} = \omega_{n}\)

Damping Ratio

\(\frac{\sqrt{R_{1} R_{2}}}{R_{2}} = \zeta\)

First Order Highpass Filter

KCL at Inverting Input

\(\frac{-V_{in}}{R_{1} + Z_{c}} - \frac{V_{out}}{R_{2}} = 0\)

\(\frac{-R_{2} V_{in}}{R_{1} + Z_{c}} = V_{out}\)

\(\frac{-R_{2} V_{in}}{R_{1} + \frac{1}{C s}} = V_{out}\)

\(\frac{-C R_{2} s}{C R_{1} s + 1} = \frac{V_{out}}{V_{in}}\)

\(- \frac{R_{2}}{R_{1}} \frac{C s}{C s + \frac{1}{R_{1}}} = \frac{V_{out}}{V_{in}}\)

Final Transfer Function

\(- \frac{R_{2}}{R_{1}} \frac{R_{1} C s}{R_{1} C s + 1} = \frac{V_{out}}{V_{in}}\)

Looking at the transfer function, we can determine the system characteristics in terms of the circuit components.

Passband Gain

\(- \frac{R_{2}}{R_{1}} = H_{o}\)

Cut-off Frequency (pole location)

\(\frac{1}{2 \pi C R_{1}} = F_{o}\)

First Order Lowpass Filter

KCL at Inverting Input

\(-\frac{V_{in}}{R_{1}} - \frac{V_{out}}{\frac{R_{2}Z_{c}}{R_{2} + Z_{c}}} = 0\)

\(\frac{V_{out}}{\frac{R_{2} Z_{c}}{R_{2} + Z_{c}}} = -\frac{V_{in}}{R_{1}}\)

\(\frac{V_{out}(R_{2} + Z_{c})}{R_{2} Z_{c}} = - \frac{V_{in}}{R_{1}}\)

\(-\frac{R_{1}(R_{2} + Z_{c})}{R_{2} Z_{c}} = \frac{V_{in}}{V_{out}}\)

\(-\frac{R_{1}(R_{2} + \frac{1}{C s})}{R_{2} \cdot \frac{1}{C s}} = \frac{V_{in}}{V_{out}}\)

\(-\frac{C s R_{1} R_{2} + R_{1}}{R_{2}} = \frac{V_{in}}{V_{out}}\)

\(-\frac{R_{1}}{R_{2}}(\frac{C R_{2} s + 1}{1}) = \frac{V_{in}}{V_{out}}\)

Final Transfer Function

\(-\frac{R_{2}}{R_{1}}(\frac{1}{CR_{2}s+1}) = \frac{V_{out}}{V_{in}}\)
